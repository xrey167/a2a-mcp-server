# Kickoff Checklist (Customer + Internal)

Use this in the kickoff call and mark each item explicitly.

## A) Business Alignment
- [ ] Primary workflow selected (`reporting`, `approval`, or `handoff`).
- [ ] Business owner named (customer-side).
- [ ] Success criteria approved (what counts as pilot success).
- [ ] Weekly decision meeting scheduled.

## B) KPI Baseline
- [ ] Current cycle time documented.
- [ ] Current manual steps documented.
- [ ] Current failure/rework rate documented.
- [ ] Baseline source and measurement method agreed.

## C) Technical Readiness
- [ ] Target connector confirmed (`odoo`, `business-central`, `dynamics`).
- [ ] API access confirmed with required permissions.
- [ ] Data mapping fields confirmed (external IDs, status mapping, ownership fields).
- [ ] Auth/security contact assigned.

## D) Environment Setup
- [ ] Workspace created.
- [ ] Roles assigned (ops lead, approver, admin).
- [ ] Connector connected.
- [ ] Health check passed.
- [ ] Initial readiness report generated.

## E) Go-Live Guardrails
- [ ] Incident owner and escalation path confirmed.
- [ ] Retry/rollback policy acknowledged.
- [ ] Communication template for incident updates approved.
- [ ] First live run approval authority confirmed.
